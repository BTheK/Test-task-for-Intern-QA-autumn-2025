# Тестовая документация микросервиса объявлений

Ниже представлены тест-кейсы для проверки API функционала микросервиса из задания 2 этапа отбора на стажировку QA в Авито

Задание №2.1

**Окружение:** https://qa-internship.avito.com

---

## TC-1: Успешное создание объявления
**Приоритет:** Critical (Блокирующий)
**Метод:** `POST`
**Endpoint:** `/api/1/item`

**Входные данные (Body):**

    {
      "sellerID": SELLER_ID,
      "name": "Test Item",
      "price": 55000,
      "statistics": { "contacts": 3, "likes": 10, "viewCount": 100 }
    }

**Шаги:**
1. Отправить POST запрос на указанный endpoint с валидным телом запроса.

**Ожидаемый результат:**
* Статус ответа: **200 OK**.
* В теле ответа приходит сообщение формата `'Сохранили объявление - {id объявления в формате UUID}'`.

**Комментарий:**
SELLER_ID - любое целое число в диапазоне 111111-999999

---

## TC-2: Получение объявления по идентификатору
**Приоритет:** High (Высокий)
**Метод:** `GET`
**Endpoint:** `/api/1/item/{id}`

**Предусловия:**
Объявление создано, ID объявления известен.

**Шаги:**
1. Отправить GET запрос, указав валидный `id` существующего объявления.

**Ожидаемый результат:**
* Статус ответа: **200 OK**.
* Тело полученного JSON содержит поля: `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.
* Значения полей совпадают с теми, что были переданы при создании.

---

## TC-3: Получение всех объявлений продавца
**Приоритет:** Medium (Средний)
**Метод:** `GET`
**Endpoint:** `/api/1/{sellerID}/item`

**Предусловия:**
Существует продавец с `sellerID`, у которого создано минимум одно объявление.

**Шаги:**
1. Отправить GET запрос с указанием `sellerID`.

**Ожидаемый результат:**
* Статус ответа: **200 OK**.
* Тело ответа содержит массив (список) объявлений.
* *Если кейсы прогоняются по порядку:* В массиве присутствует объявление, созданное ранее в TC-1.

---

## TC-4: Получение статистики по объявлению
**Приоритет:** Medium (Средний)
**Метод:** `GET`
**Endpoint:** `/api/1/statistic/{id}`

**Предусловия:**
* Объявление создано, ID известен.

**Шаги:**
1. Отправить GET запрос с указанием валидного `id`.

**Ожидаемый результат:**
* Статус ответа: **200 OK**.
* Тело ответа содержит поля: `likes`, `viewCount`, `contacts`.
* *Если кейсы прогоняются по порядку:* Значения соответствуют переданным при создании в TC-1.
